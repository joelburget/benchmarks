Getting Started

Membership

Why Join?

Anyone can view posts and comments but members enjoy these benefits:

  • Propose new benchmarks or solutions on this site.
  • Comment on other posts.
  • Collaborate with other researchers interested in software verification in a centralized location.
  • Allow other researchers to contact you easily (please note: we will share only the information you allow).

How to Join

If you don't have an account yet, please create one now. We will respond, hopefully, within a day or two, providing you with a link in your email inbox to let you create your account.

Posting Content

Post Types

Each post can be classified as a Problem, a proposed Solution to a problem, or Other (used for interesting papers, for example). To create a new Problem, see the next section on on adding new problems. To propose a new Solution to a problem, scroll to the "Actions" section of a problem and click "Add a solution to this problem."

Adding New Problems

Click on New Problem in the right-hand toolbar. This will take you to an editor where you can create benchmark problems for others to submit solutions to.

Formatting Content

Most HTML tags, such as <p>, <b>, <i>, and <pre> are allowed. CSS such as style="font-weight:bold;" is also allowed, but not recommended. We have default stylings for all page elements. Javascript is not allowed for security reasons. Additionally, LaTeX formatting can be achieved by surrounding single line formulas in double-dollar signs (for example, $$x^2$$).

Uploading Attachments to Posts

Files can be included through HTML hyperlinks, or by using the file upload button below the text field. You may upload as many files and filetypes as you like. Zipped folders and tarballs (*.zip, *.tar.gz, *.tar.bz2, or similar) will automatically be decompressed. Users will be offered the choice of looking in a tree browser to preview files one at a time, or downloading the entire archive.

Comments

Commenting is included for every post. Comments are numbered to aid discussion. The same HTML, CSS, and LaTeX that can be used in posts can be used in comments.
