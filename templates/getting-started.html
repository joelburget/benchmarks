Getting Started

Membership

Anyone can view posts and comments but members enjoy these benefits:

  • Propose new benchmarks or solutions on this site.
  • Comment on other posts.
  • Collaborate with other researchers interested in software verification in a centralized location.
  • Allow other researchers to contact you more easily (we will share only the information you allow).

Get a username

The rest of this guide assumes you have a username. If you don't have one yet please email Professor Bruce Weide at weide (dot) 1 (at) osu (dot) edu to set up an account. Please include your name, organization (if any), and why you would like to participate.

Set your information

After you sign in for the first time you can click on the link to your profile in the top right (it should be your name). You can add a short bio about yourself and set your email to be shown to other users. We will never show your email to visitors who aren't signed in

Post

Once your account is set up the next step is to post something. Click on "new post" in the toolbar which will take you to an editor. Most html tags such as <p>, <b>, and <code> are allowed. CSS is also allowed but not recommended. Javascript is of course not allowed. For more information on how we sanitize posts see here, although everything will usually just work. Files can be included through hyperlinks or by using the file upload button below the text field. Each post can be classified as a Problem, Realization, Verification, or Other. Each type of post except Problem can be linked to the post it is in response to. For instance a verification would be linked to the realization it verifies. See other posts on the site for examples.

Comments

Comments are included for every post. The same html and css that can be used in posts can be used in comments.
